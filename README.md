# @lemon/kanban

## ✨ 주요 기능

| 기능 | 설명 |
|------|------|
| 📋 **칸반 보드** | 커스터마이징 가능한 다열 구성 (기본: TO-DO / IN PROGRESS / DONE) |
| 🏷️ **해시태그** | `#태그` 형식 입력 · 클릭 한 번으로 필터링 |
| 🔍 **태그별 그룹 보기** | 완료 카드 등을 태그 기준으로 묶어서 조회 |
| ➕ **컬럼 커스터마이징** | 보드에서 바로 추가 · 설정에서 삭제 / 순서 변경 · Flush 가능 여부 지정 |
| 🗃️ **보관 & 아카이브** | 보관 가능 컬럼의 카드를 일괄 아카이브, 월별로 구조화 저장 |
| 📜 **아카이브 히스토리** | 보드 내 아카이브 뷰에서 검색 · 월별 / 컬럼별 / 태그별 필터 |
| ☑️ **체크리스트** | 카드 내 항목 체크리스트 · 카드 미리보기에 완료율 배지 표시 |
| 🔎 **전역 검색** | 보드 헤더 검색창 — 제목 / 내용 / 태그 실시간 필터 |
| 🔃 **카드 정렬** | 생성일 / 마감일 / 우선순위 / 제목 기준 정렬 (클릭으로 오름·내림차순 전환) |
| 📅 **마감 임박 뷰** | 마감일 기준 그룹 보기 — 기한 초과 / 오늘 / 7일 / 30일 / 이후 |
| 🖱️ **드래그 앤 드롭** | 카드를 열 간에 자유롭게 이동 |
| 📁 **vault-native 저장** | 카드 1개 = `.md` 파일 1개, Obsidian에서 직접 열람 가능 |
| 🔁 **자동 새로고침** | vault 파일 변경 시 뷰 자동 갱신 |

## 📦 설치

### BRAT 사용 (권장)

> 업데이트가 자동으로 적용됩니다.

1. Obsidian 커뮤니티 플러그인에서 **BRAT** 설치 및 활성화
2. BRAT 설정 → **Add Beta plugin from GitHub**
3. `Lemon-Obsidian/kanban` 입력
4. Obsidian 재시작 후 `@lemon/kanban` 플러그인 활성화

### 수동 설치

1. [최신 릴리즈](https://github.com/Lemon-Obsidian/kanban/releases/latest)에서 `main.js`, `manifest.json`, `styles.css` 다운로드
2. vault 내 `.obsidian/plugins/lemon-kanban/` 폴더에 복사
3. Obsidian 재시작 후 설정 → 커뮤니티 플러그인에서 활성화

## 🗂️ 동작 방식

### 파일 저장 구조

카드는 설정한 보드 폴더 아래에 **컬럼 ID = 서브폴더** 방식으로 저장됩니다.
컬럼을 추가/삭제하면 폴더도 함께 생성됩니다.

```
Kanban/                     ← 설정에서 변경 가능 (기본값: "Kanban")
├── todo/                   ← TO-DO 컬럼
│   ├── 디자인-시안-검토.md
│   └── API-명세-작성.md
├── doing/                  ← IN PROGRESS 컬럼
│   └── 로그인-기능-구현.md
├── done/                   ← DONE 컬럼 (flushable)
│   └── DB-스키마-설계.md
├── review/                 ← 커스텀 컬럼 (직접 추가한 경우)
│   └── 코드-리뷰-요청.md
└── _archive/               ← Flush된 카드가 월별로 저장
    ├── 2026-03/
    │   └── DB-스키마-설계.md   ← flushedAt, flushedFrom 메타 추가됨
    └── 2026-02/
        └── 이전-완료-카드.md
```

### 카드 파일 형식

각 카드는 frontmatter + 마크다운 본문으로 구성된 일반 `.md` 파일입니다.
vault에서 직접 열어 편집해도 플러그인에 자동 반영됩니다.

```markdown
---
tags:
  - 업무
  - 프로젝트A
due: 2024-03-15
priority: high
created: 2024-01-01T10:00:00.000Z
---

# 카드 제목

카드 본문 내용 (선택 사항)
```

### 뷰 구조

```
@lemon/kanban
├── Kanban Board  (리본: columns 아이콘)
│   ├── 검색창 (제목 / 내용 / 태그 실시간 검색)
│   ├── 정렬 버튼 (생성일 / 마감일 / 우선순위 / 제목)
│   ├── 태그 필터 바
│   ├── [TO-DO]  [IN PROGRESS]  [DONE]  ...커스텀 컬럼  [+ 컬럼 추가]
│   │    ↕ 드래그 앤 드롭으로 카드 이동
│   │    + 버튼 → 카드 추가 모달 (체크리스트 포함)
│   │    🗃 보관 (N) → 아카이브로 일괄 보관 (보관 가능 컬럼만)
│   │    ··· → 컬럼 삭제
│   │    우클릭 → 편집 / 이동 / 삭제
│   └── (클릭 → 카드 편집 모달)
│
├── 마감 임박 뷰  (보드 헤더 [마감 임박] 버튼)
│   └── 기한 초과 / 오늘 / 이번 주 / 이번 달 / 이후 그룹
│
├── 아카이브  (보드 헤더 [아카이브] 버튼)
│   └── 검색 · 월별 / 컬럼별 / 태그별 필터
│
└── 태그별 보기  (리본: tag 아이콘)
    ├── 상태 필터: 전체 / 각 컬럼
    └── 태그 그룹 (클릭으로 접기/펼치기)
         └── 카드 목록 (상태 배지 · 마감일 · 우선순위)
```

## 📖 사용법

### 보드 열기

- 왼쪽 리본의 **columns 아이콘** 클릭
- 또는 커맨드 팔레트 (`Ctrl/Cmd+P`) → `Kanban 보드 열기`

### 카드 추가

1. 원하는 열 상단의 **`+` 버튼** 클릭
2. 모달에서 입력

| 필드 | 설명 | 필수 |
|------|------|:----:|
| 제목 | 카드 제목 | ✅ |
| 태그 | `#업무 #프로젝트` 형식으로 입력 | |
| 마감일 | 날짜 선택 (지난 날짜는 빨간색 강조) | |
| 우선순위 | 낮음 🔵 / 중간 / 높음 🔴 / ASAP 🚨 (기본: 중간) | |
| 내용 | 자유 텍스트 | |

3. **추가** 버튼 또는 제목 입력 후 `Enter`

### 카드 이동

| 방법 | 설명 |
|------|------|
| 드래그 앤 드롭 | 카드를 다른 열 위로 끌어다 놓기 |
| 우클릭 메뉴 | 이동할 컬럼 선택 |

### 태그 필터

- 보드 상단 필터 바에서 태그 클릭 → 해당 태그 카드만 표시
- 카드의 `#태그` 직접 클릭도 동일
- **전체** 버튼으로 필터 해제

### 태그별 그룹 보기

- 리본의 **tag 아이콘** 클릭 또는 커맨드 팔레트 → `태그별 보기 열기`
- 상단 필터로 **전체** 또는 특정 컬럼 선택
- 태그 그룹 헤더 클릭으로 접기 / 펼치기
- 카드 클릭으로 편집 가능

### 컬럼 관리

**보드에서 바로 추가**: 보드 오른쪽 끝의 **`+ 컬럼 추가`** 클릭 → 이름 · Flush 가능 여부 입력

**설정에서 관리**: 설정 → 플러그인 옵션 → `@lemon/kanban` → **컬럼 관리**

| 동작 | 방법 |
|------|------|
| 추가 | 보드 오른쪽 `+ 컬럼 추가` 또는 설정 탭 (폴더명 자동 생성) |
| 삭제 | 컬럼 헤더 hover 시 나타나는 `···` 클릭 → 컬럼 삭제 (카드가 있으면 삭제 불가) |
| 삭제 | 🗑️ 버튼 (카드가 있으면 삭제 불가 — 먼저 카드 이동/삭제) |
| 순서 변경 | ↑ / ↓ 버튼으로 보드 표시 순서 조정 |
| 이름 수정 | 텍스트 필드 직접 수정 (폴더명은 변경되지 않음) |
| 보관 가능 | 토글로 활성화 — DONE은 기본값으로 활성화, 신규 컬럼은 추가 시 선택 |
| 삭제 | 컬럼 헤더의 `···` → 컬럼 삭제 (카드가 있으면 삭제 불가) |

### 전역 검색

보드 헤더 검색창에 입력하면 모든 컬럼의 카드를 제목 / 내용 / 태그 기준으로 실시간 필터링합니다.

### 카드 정렬

보드 헤더의 정렬 버튼으로 정렬 기준 선택:

| 버튼 | 기준 |
|------|------|
| 생성일 | 최신 순 (기본값) |
| 마감일 | 마감일 오름차순 |
| 우선순위 | ASAP → 높음 → 중간 → 낮음 |
| 제목 | 가나다 순 |

같은 버튼을 다시 누르면 오름/내림차순이 전환됩니다 (↓/↑ 표시).

### 체크리스트

카드 추가/편집 모달에 **체크리스트** 섹션이 있습니다.

- `+ 항목 추가` 클릭 또는 항목 입력 후 `Enter` → 새 항목 추가
- `Backspace` (빈 항목) → 항목 삭제
- 체크박스로 완료 처리 → 카드 미리보기에 `☑ 2/5` 형식으로 진행률 표시
- vault `.md` 파일에 `- [ ] 항목` / `- [x] 항목` 형식으로 저장 (Obsidian 호환)

### 마감 임박 뷰

보드 헤더의 **마감 임박** 버튼을 클릭하면 마감일이 설정된 카드를 기간별로 그룹화하여 표시합니다.

| 그룹 | 기준 |
|------|------|
| 기한 초과 | 오늘 이전 (빨간색 강조) |
| 오늘 | 오늘 마감 |
| 이번 주 (7일 이내) | 내일 ~ 7일 후 |
| 이번 달 (30일 이내) | 8일 후 ~ 30일 후 |
| 이후 | 30일 초과 |

카드 클릭 시 편집 모달이 열립니다.

### 보관 & 아카이브

**보관**은 컬럼의 카드를 일괄 아카이브로 이동하는 기능입니다. 업무 히스토리를 깔끔하게 보존할 수 있습니다.

1. 보관 가능 컬럼의 헤더에 **`🗃 보관 (N)`** 버튼이 표시됨
2. 버튼 클릭 → 확인 다이얼로그 (카드 수 표시)
3. **보관하기** 클릭 시 카드가 `_archive/YYYY-MM/` 폴더로 이동 (보관 날짜 · 원본 컬럼 메타데이터 자동 추가)

**아카이브 히스토리 보기**

1. 보드 상단의 **아카이브** 버튼 클릭
2. 필터 조합으로 원하는 카드 탐색:

| 필터 | 설명 |
|------|------|
| 검색창 | 제목 · 내용 · 태그 전문 검색 |
| 월 | Flush된 월 기준 필터 |
| 컬럼 | 원본 컬럼 기준 필터 |
| 태그 | 태그 기준 필터 |

3. 카드 클릭으로 상세 편집 가능 (메모 추가 등)
4. **← 보드로 돌아가기** 버튼으로 복귀

## 🛠️ 개발

```bash
# 의존성 설치
pnpm install

# 개발 모드 (파일 변경 감지 + 자동 빌드)
pnpm dev

# 프로덕션 빌드
pnpm build
```

빌드 후 `main.js`, `manifest.json`, `styles.css`를
vault의 `.obsidian/plugins/lemon-kanban/`에 복사하면 즉시 반영됩니다.

### 릴리즈

버전 태그를 push하면 GitHub Actions가 자동으로 빌드 후 릴리즈를 생성합니다.

```bash
# package.json / manifest.json 버전 업데이트 후
git tag 1.1.0
git push origin 1.1.0
```

### 프로젝트 구조

```
src/
├── main.ts          # 플러그인 진입점 · 뷰 등록 · vault 이벤트 처리
├── types.ts         # KanbanCard · KanbanColumn · KanbanSettings 타입
├── utils.ts         # slugify · parseTags · formatTags
├── FileManager.ts   # 카드 파일 CRUD (생성 · 읽기 · 이동 · 삭제)
├── KanbanView.ts    # 칸반 보드 뷰 (3열+ · 드래그앤드롭 · 태그 필터)
├── CardModal.ts     # 카드 추가 / 편집 모달
└── TagGroupView.ts  # 태그별 그룹 보기 뷰
styles.css           # 플러그인 스타일 (Obsidian 자동 로드)
manifest.json        # 플러그인 메타 (id: lemon-kanban)
```
