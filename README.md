# Kanban with Hashtags

Obsidian용 칸반 보드 플러그인. 해시태그(#tag) 기반 필터링과 태그별 카드 그룹 보기를 지원합니다.

## 주요 기능

- **칸반 보드** — 커스터마이징 가능한 다열 구성 (기본: TO-DO / IN PROGRESS / DONE)
- **컬럼 관리** — 설정에서 컬럼 추가/삭제/순서 변경 가능
- **UI로 카드 추가** — 각 열 상단의 `+` 버튼으로 모달 열기
- **해시태그 지원** — `#태그` 형식으로 태그 입력 및 필터링
- **태그별 그룹 보기** — 완료된 카드(또는 전체)를 태그별로 묶어서 확인
- **드래그 앤 드롭** — 열 간 카드 이동
- **메타데이터** — 마감일, 우선순위(낮음/보통/높음), 내용
- **폴더 기반 저장** — 카드 1개 = `.md` 파일 1개 (vault에서 직접 열람 가능)
- **자동 새로고침** — vault 파일 변경 시 뷰 자동 갱신

## 설치

### BRAT 사용 (권장)

1. Obsidian 커뮤니티 플러그인에서 **BRAT** 설치 및 활성화
2. BRAT 설정 → **Add Beta plugin**
3. 이 저장소 URL 입력
4. Obsidian을 재시작하고 플러그인 활성화

### 수동 설치

1. [최신 릴리즈](../../releases/latest)에서 `main.js`, `manifest.json`, `styles.css` 다운로드
2. vault 내 `.obsidian/plugins/lemon-kanban/` 폴더에 복사
3. Obsidian 재시작 후 설정 → 커뮤니티 플러그인에서 활성화

## 사용법

### 보드 열기

- 왼쪽 리본의 **칸반 아이콘** 클릭
- 또는 커맨드 팔레트 → `Kanban 보드 열기`

### 컬럼 추가 / 삭제 / 순서 변경

설정 → 플러그인 옵션 → **Kanban with Hashtags** → **컬럼 관리**

- **추가** — 이름 입력 후 추가 버튼 (폴더명은 자동 생성)
- **삭제** — 휴지통 아이콘 (카드가 있으면 삭제 불가, 먼저 카드를 이동하거나 삭제)
- **순서 변경** — ↑ / ↓ 버튼으로 컬럼 순서 조정

### 카드 추가

1. 원하는 열 상단의 **`+` 버튼** 클릭
2. 모달에서 입력:
   - **제목** (필수)
   - **태그** — `#업무 #프로젝트` 형식으로 공백으로 구분하여 입력
   - **마감일** — 날짜 선택
   - **우선순위** — 없음 / 낮음 / 보통 / 높음
   - **내용** — 자유 텍스트
3. **추가** 버튼 또는 제목 입력 후 `Enter`

### 카드 이동

| 방법 | 설명 |
|------|------|
| 드래그 앤 드롭 | 카드를 다른 열로 끌어다 놓기 |
| 우클릭 메뉴 | 이동할 열 선택 |

### 카드 편집 / 삭제

- **클릭** — 편집 모달 열기
- **우클릭** — 편집, 이동, 삭제 메뉴

### 태그 필터

- 보드 상단 필터 바에서 태그 버튼 클릭 → 해당 태그 카드만 표시
- 카드의 `#태그` 직접 클릭도 동일하게 동작
- **전체** 버튼으로 필터 해제

### 태그별 그룹 보기

- 리본의 **태그 아이콘** 클릭 또는 커맨드 팔레트 → `태그별 보기 열기`
- 상단에서 상태 필터 선택: **전체** 또는 각 컬럼 (추가한 커스텀 컬럼도 표시됨)
- 태그 그룹 헤더 클릭으로 접기/펼치기
- 카드 클릭으로 편집 가능

### 우선순위 표시

카드 왼쪽 컬러 바로 구분:

| 색상 | 우선순위 |
|------|---------|
| 🔴 빨간색 | 높음 |
| 🟡 노란색 | 보통 |
| 🔵 파란색 | 낮음 |

### 마감일 강조

오늘 날짜가 지난 카드의 마감일은 빨간색으로 표시됩니다 (DONE 열 제외).

## 파일 저장 구조

카드는 설정한 보드 폴더 아래에 상태별 서브폴더로 저장됩니다.

```
Kanban/          ← 설정에서 변경 가능 (기본값: "Kanban")
├── todo/
│   └── 내-카드-제목.md
├── doing/
│   └── 진행중-카드.md
└── done/
    └── 완료된-카드.md
```

각 카드 파일 형식:

```markdown
---
tags:
  - 업무
  - 프로젝트
due: 2024-03-15
priority: high
created: 2024-01-01T10:00:00.000Z
---

# 카드 제목

카드 내용 (선택 사항)
```

vault에서 직접 파일을 열어 편집해도 플러그인에 자동 반영됩니다.

## 설정

설정 → 플러그인 옵션 → **Kanban with Hashtags**

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 보드 폴더 | 카드 파일을 저장할 최상위 폴더 경로 | `Kanban` |
| 컬럼 관리 | 컬럼 추가/삭제/순서 변경, 이름 수정 | TO-DO, IN PROGRESS, DONE |

## 개발

```bash
# 의존성 설치
pnpm install

# 개발 모드 (파일 변경 감지)
pnpm dev

# 프로덕션 빌드
pnpm build
```

빌드 결과물(`main.js`)을 vault의 플러그인 폴더에 복사하면 즉시 반영됩니다.

### 릴리즈 배포

버전 태그를 push하면 GitHub Actions가 자동으로 빌드 및 릴리즈를 생성합니다:

```bash
git tag 1.0.1
git push origin 1.0.1
```
