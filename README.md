## ✨ 주요 기능

| 기능 | 설명 |
|------|------|
| 🗂️ **멀티 보드** | 여러 칸반 보드를 독립적으로 생성 · 보드별 폴더 경로 · 헤더 드롭다운으로 즉시 전환 |
| 📋 **칸반 보드** | 커스터마이징 가능한 다열 구성 (기본: TO-DO / IN PROGRESS / DONE) |
| 🏷️ **해시태그** | `#태그` 형식 입력 · 자동완성 드롭다운 · 클릭 한 번으로 필터링 · 태그별 카드 수 뱃지 |
| 🔍 **전역 검색** | 제목 / 내용 / 태그 실시간 필터 · 매칭 텍스트 노란색 하이라이트 |
| 🔎 **태그별 그룹 보기** | 모든 카드를 태그 기준으로 묶어서 조회 |
| 📊 **보드 통계** | 헤더에 전체 · 완료 · 기한 초과 카드 수 실시간 표시 |
| ➕ **컬럼 커스터마이징** | 보드에서 바로 추가 / 이름 변경(폴더 자동 리네임) / 드래그로 순서 변경 · Flush 가능 여부 지정 |
| 🗃️ **보관 & 아카이브** | 보관 가능 컬럼의 카드를 일괄 아카이브, 월별로 구조화 저장 |
| 📜 **아카이브 히스토리** | 보드 내 아카이브 뷰에서 검색 · 월별 / 컬럼별 / 태그별 필터 |
| ☑️ **체크리스트** | 카드 내 항목 체크리스트 · 완료율 배지 표시 |
| 🔢 **우선순위** | 낮음 🔵 / 중간 🟡 / 높음 🔴 / ASAP 🚨 · 카드에 인라인 chip으로 표시 |
| 🔁 **반복 카드** | 매일 / 매주 / 매월 설정 시 완료 이동 때 TO-DO에 자동 생성 · 마감일 자동 계산 |
| 🔗 **내부 링크** | 카드 내용의 `[[노트명]]` · `[[노트명\|별칭]]` 클릭 시 해당 노트 바로 열기 |
| 📅 **마감 임박 뷰** | 마감일 기준 그룹 보기 — 기한 초과 / 오늘 / 7일 / 30일 / 이후 |
| 🕐 **상대적 날짜** | 카드 하단에 생성일 · 수정일을 "N일 전" 형식으로 표시 |
| 🔃 **카드 정렬** | 생성일 / 마감일 / 우선순위 / 제목 기준 정렬 · 오름/내림차순 전환 |
| 🖱️ **드래그 앤 드롭** | 카드 · 컬럼 모두 드래그로 자유롭게 이동 |
| 📁 **vault-native 저장** | 카드 1개 = `.md` 파일 1개, Obsidian에서 직접 열람 가능 |
| 🔁 **자동 새로고침** | vault 파일 변경 시 뷰 자동 갱신 |

## 📦 설치

### BRAT 사용 (권장)

> 업데이트가 자동으로 적용됩니다.

1. Obsidian 커뮤니티 플러그인에서 **BRAT** 설치 및 활성화
2. BRAT 설정 → **Add Beta plugin from GitHub**
3. `Lemon-Obsidian/kanban` 입력
4. Obsidian 재시작 후 `@lemon/kanban` 플러그인 활성화

### 수동 설치

1. [최신 릴리즈](https://github.com/Lemon-Obsidian/kanban/releases/latest)에서 `main.js`, `manifest.json`, `styles.css` 다운로드
2. vault 내 `.obsidian/plugins/lemon-kanban/` 폴더에 복사
3. Obsidian 재시작 후 설정 → 커뮤니티 플러그인에서 활성화

## 🗂️ 동작 방식

### 파일 저장 구조

카드는 각 보드의 폴더 아래에 **컬럼 ID = 서브폴더** 방식으로 저장됩니다.
보드를 여러 개 만들면 각자 독립적인 폴더를 가집니다.

```
Kanban/                       ← 보드 루트 폴더 (설정에서 변경 가능)
├── todo/                     ← TO-DO 컬럼
│   ├── 디자인-시안-검토.md
│   └── API-명세-작성.md
├── in-progress/              ← IN PROGRESS 컬럼
│   └── 로그인-기능-구현.md
├── done/                     ← DONE 컬럼 (flushable)
│   └── DB-스키마-설계.md
└── _archive/                 ← Flush된 카드가 월별로 저장
    └── 2026-03/
        └── DB-스키마-설계.md  ← flushedAt, flushedFrom 메타 추가됨

WorkKanban/                   ← 별도 보드 폴더 (멀티 보드)
├── todo/
└── done/
```

### 카드 파일 형식

각 카드는 frontmatter + 마크다운 본문으로 구성된 일반 `.md` 파일입니다.
vault에서 직접 열어 편집해도 플러그인에 자동 반영됩니다.

```markdown
---
tags:
  - 업무
  - 프로젝트A
due: 2026-03-15
priority: high
recur: weekly
created: 2026-01-01T10:00:00.000Z
---

# 카드 제목

카드 본문 내용. [[관련 노트]]를 링크로 연결할 수 있습니다.

- [ ] 체크리스트 항목 1
- [x] 체크리스트 항목 2 (완료)
```

### 뷰 구조

```
@lemon/kanban
├── Kanban Board  (리본: columns 아이콘)
│   ├── 보드 전환 드롭다운 + [+ 보드 추가]
│   ├── [📅 마감 임박]  [🗃 아카이브]
│   ├── 검색창 (매칭 텍스트 하이라이트)
│   ├── 정렬 버튼 (생성일 / 마감일 / 우선순위 / 제목)
│   ├── 태그 필터 바 (태그별 카드 수 뱃지)
│   ├── N 전체 · N 완료 · N 기한 초과  ← 통계 바
│   ├── [TO-DO]  [IN PROGRESS]  [DONE]  ...커스텀 컬럼  [+ 컬럼 추가]
│   │    ↕ 드래그 앤 드롭으로 카드 · 컬럼 이동
│   │    + 버튼 → 카드 추가 모달
│   │    🗃 보관 (N) → 아카이브로 일괄 보관
│   │    ··· → 컬럼 이름 변경 / 삭제
│   │    ── 카드 ──
│   │       제목 [우선순위 chip]
│   │       🔁 매주  (반복 카드)
│   │       📅 YYYY-MM-DD까지
│   │       내용 미리보기 ([[링크]] 클릭 가능)
│   │       체크리스트 (☑ 2/5, [[링크]] 클릭 가능)
│   │       #태그 #태그
│   │       생성 N일 전 · 수정 N시간 전
│   │    우클릭 → 편집 / 이동 / 삭제
│   └── (카드 hover → ✎ 편집 / ✕ 삭제 버튼)
│
├── 마감 임박 뷰
│   └── 기한 초과 / 오늘 / 이번 주 / 이번 달 / 이후 그룹
│
├── 아카이브
│   └── 검색 · 월별 / 컬럼별 / 태그별 필터
│
└── 태그별 보기  (리본: tag 아이콘)
    ├── 상태 필터: 전체 / 각 컬럼
    └── 태그 그룹 (클릭으로 접기/펼치기)
         └── 카드 목록 (상태 배지 · 마감일 · 우선순위)
```

## 📖 사용법

### 보드 열기

- 왼쪽 리본의 **columns 아이콘** 클릭
- 또는 커맨드 팔레트 (`Ctrl/Cmd+P`) → `Kanban 보드 열기`

### 멀티 보드

보드 헤더의 **드롭다운**에서 보드를 전환합니다. `+` 버튼으로 새 보드를 추가하고, 설정 탭에서 각 보드의 이름과 폴더를 관리합니다.

### 카드 추가

1. 원하는 열 상단의 **`+` 버튼** 클릭
2. 모달에서 입력

| 필드 | 설명 | 필수 |
|------|------|:----:|
| 제목 | 카드 제목 | ✅ |
| 태그 | `#업무 #프로젝트` 형식 · 입력 중 기존 태그 자동완성 드롭다운 | |
| 마감일 | 날짜 선택 (지난 날짜는 빨간색 강조) | |
| 우선순위 | 🔵 낮음 / 🟡 중간 / 🔴 높음 / 🚨 ASAP (기본: 중간) | |
| 반복 | 반복 없음 / 매일 / 매주 / 매월 | |
| 내용 | 자유 텍스트 (`[[노트명]]`으로 내부 링크 가능) | |
| 체크리스트 | `+ 항목 추가` 또는 항목 입력 후 `Enter` | |

3. **추가** 버튼 또는 제목 입력 후 `Enter`

### 반복 카드

카드에 반복 주기를 설정하면 완료 컬럼(DONE 등)으로 이동 시 **첫 번째 TO-DO 컬럼에 자동으로 새 카드가 생성**됩니다.

| 설정 | 동작 |
|------|------|
| 매일 | 마감일 +1일로 새 카드 생성 |
| 매주 | 마감일 +7일로 새 카드 생성 |
| 매월 | 마감일 +1개월로 새 카드 생성 |

마감일이 없는 반복 카드는 마감일 없이 새 카드가 생성됩니다.
반복 카드에는 카드에 🔁 매일/매주/매월 배지가 표시됩니다.

### 내부 링크

카드 내용 또는 체크리스트 항목에 `[[노트명]]` 또는 `[[노트명|표시텍스트]]` 형식으로 입력하면 클릭 가능한 링크로 렌더링됩니다.

### 태그 자동완성

태그 입력란에 `#`을 입력하면 기존 카드에서 수집된 태그 목록이 드롭다운으로 표시됩니다.

- `↑` / `↓` — 항목 탐색
- `Enter` — 선택
- `Escape` — 닫기
- 여러 태그를 공백으로 구분해 연속 입력 가능

### 카드 이동

| 방법 | 설명 |
|------|------|
| 드래그 앤 드롭 | 카드를 다른 열 위로 끌어다 놓기 |
| 우클릭 메뉴 | 이동할 컬럼 선택 |

### 전역 검색

보드 헤더 검색창에 입력하면 모든 컬럼의 카드를 제목 / 내용 / 태그 기준으로 실시간 필터링합니다.
매칭된 텍스트는 노란색으로 하이라이트됩니다.

`#태그명` 형식으로 검색하면 태그 텍스트도 하이라이트됩니다.

### 태그 필터

- 보드 상단 필터 바에서 태그 클릭 → 해당 태그 카드만 표시 (각 버튼에 카드 수 뱃지 표시)
- 카드의 `#태그` 직접 클릭도 동일
- **전체** 버튼으로 필터 해제

### 카드 정렬

보드 헤더의 정렬 버튼으로 정렬 기준 선택:

| 버튼 | 기준 |
|------|------|
| 생성일 | 최신 순 (기본값) |
| 마감일 | 마감일 오름차순 |
| 우선순위 | ASAP → 높음 → 중간 → 낮음 |
| 제목 | 가나다 순 |

같은 버튼을 다시 누르면 오름/내림차순이 전환됩니다 (↓/↑ 표시).

### 체크리스트

카드 추가/편집 모달에 **체크리스트** 섹션이 있습니다.

- `+ 항목 추가` 클릭 또는 항목 입력 후 `Enter` → 새 항목 추가
- `Backspace` (빈 항목) → 항목 삭제
- 체크박스로 완료 처리 → 카드 미리보기에 `☑ 2/5` 형식으로 진행률 표시
- vault `.md` 파일에 `- [ ] 항목` / `- [x] 항목` 형식으로 저장 (Obsidian 호환)

### 마감 임박 뷰

보드 헤더의 **마감 임박** 버튼을 클릭하면 마감일이 설정된 카드를 기간별로 그룹화하여 표시합니다.

| 그룹 | 기준 |
|------|------|
| 기한 초과 | 오늘 이전 (빨간색 강조) |
| 오늘 | 오늘 마감 |
| 이번 주 (7일 이내) | 내일 ~ 7일 후 |
| 이번 달 (30일 이내) | 8일 후 ~ 30일 후 |
| 이후 | 30일 초과 |

### 컬럼 관리

**보드에서 추가**: 보드 오른쪽 끝의 **`+ 컬럼 추가`** 클릭 → 이름 · Flush 가능 여부 입력

**보드에서 순서 변경**: 컬럼 헤더를 드래그해서 순서 조정

**설정에서 관리**: 설정 → 플러그인 옵션 → `@lemon/kanban` → 보드 섹션

| 동작 | 방법 |
|------|------|
| 추가 | 보드 오른쪽 `+ 컬럼 추가` 또는 설정 탭 |
| 이름 변경 | 컬럼 헤더 hover → `···` → 이름 변경 (폴더명 자동 리네임) |
| 삭제 | `···` → 컬럼 삭제 (카드가 있으면 삭제 불가 — 먼저 카드 이동/삭제) |
| 순서 변경 | 컬럼 헤더 드래그 또는 설정 탭 ↑/↓ 버튼 |
| Flush 가능 | 설정 탭 토글로 활성화 |

### 보관 & 아카이브

**보관**은 컬럼의 카드를 일괄 아카이브로 이동하는 기능입니다. 업무 히스토리를 깔끔하게 보존할 수 있습니다.

1. 보관 가능 컬럼의 헤더에 **`🗃 보관 (N)`** 버튼이 표시됨
2. 버튼 클릭 → 확인 다이얼로그 (카드 수 표시)
3. **보관하기** 클릭 시 카드가 `_archive/YYYY-MM/` 폴더로 이동 (보관 날짜 · 원본 컬럼 메타데이터 자동 추가)

**아카이브 히스토리 보기**

1. 보드 상단의 **아카이브** 버튼 클릭
2. 필터 조합으로 원하는 카드 탐색:

| 필터 | 설명 |
|------|------|
| 검색창 | 제목 · 내용 · 태그 전문 검색 |
| 월 | Flush된 월 기준 필터 |
| 컬럼 | 원본 컬럼 기준 필터 |
| 태그 | 태그 기준 필터 |

3. 카드 클릭으로 상세 편집 가능 (메모 추가 등)
4. **← 보드로 돌아가기** 버튼으로 복귀

### 태그별 그룹 보기

- 리본의 **tag 아이콘** 클릭 또는 커맨드 팔레트 → `태그별 보기 열기`
- 상단 필터로 **전체** 또는 특정 컬럼 선택
- 태그 그룹 헤더 클릭으로 접기 / 펼치기
- 카드 클릭으로 편집 가능

### 보드 설정

각 보드별로 독립적인 설정 섹션이 표시됩니다.

| 항목 | 설명 |
|------|------|
| 보드 이름 | 헤더 드롭다운에 표시되는 이름 |
| 폴더 경로 | 카드 파일을 저장할 vault 내 폴더 경로 |
| 컬럼 관리 | 컬럼 추가 / 삭제 / 순서 변경 / Flush 가능 여부 |
| 보드 삭제 | 보드 설정 삭제 (파일은 유지됨) |

### 전역 설정

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 마감 임박 기간 | 마감 임박 뷰에서 표시할 기간 (일 단위) | 1, 7, 30 |

## 🛠️ 개발

```bash
# 의존성 설치
npm install

# 개발 모드 (파일 변경 감지 + 자동 빌드)
npm run dev

# 프로덕션 빌드
npm run build
```

빌드 후 `main.js`, `manifest.json`, `styles.css`를
vault의 `.obsidian/plugins/lemon-kanban/`에 복사하면 즉시 반영됩니다.

### 릴리즈

버전 태그를 push하면 GitHub Actions가 자동으로 빌드 후 릴리즈를 생성합니다.

```bash
# package.json / manifest.json 버전 업데이트 후
git tag v1.8.2
git push origin v1.8.2
```
